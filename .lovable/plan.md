
# תיקון המרת קישורים קצרים בכלי "טקסט חופשי"

## מצב נוכחי
| כלי | קישורים מלאים | קישורים קצרים |
|------|--------------|---------------|
| Google Sheets | עובד | עובד |
| טקסט חופשי | עובד | **לא עובד** |
| קובץ אקסל | עובד | לא עובד |

## הבעיה
קישורים קצרים כמו `https://s.lazada.co.th/s.Mwze9?cc` לא מכילים את ה-Product ID. 
צריך לעקוב אחרי ה-redirect לקבל את הכתובת המלאה שממנה אפשר לחלץ ID.

## פתרון

### 1. יצירת Edge Function חדשה: `resolve-short-links`
**קובץ:** `supabase/functions/resolve-short-links/index.ts`

פונקציה שמקבלת רשימת קישורים קצרים ומחזירה את הכתובות המלאות:
- עוקבת אחרי redirects (עד 5 רמות)
- תומכת בדומיינים:
  - `s.lazada.co.th`
  - `c.lazada.co.th`
  - `s.click.aliexpress.com`
  - `a.aliexpress.com`
- מעבדת ב-batches של 5 למניעת timeout

**קלט/פלט:**
```text
קלט: { urls: ["https://s.lazada.co.th/s.Mwze9?cc", "https://s.click.aliexpress.com/e/xxx"] }
פלט: { 
  success: true,
  resolved: {
    "https://s.lazada.co.th/s.Mwze9?cc": "https://www.lazada.co.th/products/tcl-...-i4550225510-s..."
  }
}
```

### 2. עדכון קובץ: `supabase/config.toml`
הוספת הגדרה ל-Edge Function החדשה:
```toml
[functions.resolve-short-links]
verify_jwt = false
```

### 3. עדכון קובץ: `src/components/admin/LinkConverter.tsx`

#### א. הוספת פונקציית עזר לזיהוי קישורים קצרים
```typescript
function isShortLink(url: string): boolean {
  const shortLinkDomains = [
    's.lazada.co.th',
    'c.lazada.co.th',
    's.click.aliexpress.com',
    'a.aliexpress.com',
  ];
  return shortLinkDomains.some(domain => url.includes(domain));
}
```

#### ב. עדכון פונקציית `handleConvert`
בתחילת התהליך, לפני המרה:
1. זיהוי קישורים קצרים שאין להם Product ID
2. קריאה ל-`resolve-short-links` לפתירת הקישורים
3. החלפת קישורים קצרים בכתובות מלאות
4. המשך תהליך ההמרה הרגיל

#### ג. הוספת הודעת התקדמות
תצוגת "פותר קישורים קצרים..." לפני "ממיר קישורים..."
