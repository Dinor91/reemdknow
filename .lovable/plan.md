
# גישה B — יום עבודה שמתחיל ב-12:00

## הרעיון
"היום" = כל מוצר שנוצר מ-12:00 של אתמול ועד 12:00 של היום. כך סריקת לילה (שרצה בין 00:00 ל-06:00) נכנסת ל"היום" של הבוקר, ומתיישרת עם מחזור העבודה שלך (סריקה בלילה → פרסום בבוקר).

## שינויים בקובץ `src/components/admin/v2/ScoutDraftsTab.tsx`

### 1. פונקציית עזר משותפת — `workdayStartFor(date)`
מחזירה את תחילת "יום העבודה" עבור תאריך נתון:
- אם השעה הנוכחית ≥ 12:00 → תחילת יום העבודה = היום ב-12:00
- אם השעה הנוכחית < 12:00 → תחילת יום העבודה = אתמול ב-12:00

### 2. עדכון המדד `draftsLast24h` (שורות 119-122)
במקום חלון מתגלגל של 24 שעות, יספור רק מוצרים שנוצרו אחרי `workdayStartFor(now)`. ישונה גם שם המשתנה ל-`draftsToday` לבהירות.

### 3. עדכון לוגיקת הקיבוץ `dayGroups` (שורות 133-166)
במקום לקבץ לפי `getDate()` קלנדרי, נחשב לכל מוצר את "יום העבודה" שלו:
- מוצר שנוצר ב-24/5 18:42 → שייך ליום העבודה של 25/5 (כי 18:42 > 12:00 של 24/5, ופחות מ-12:00 של 25/5)
- המפתח לקיבוץ הופך ל-workday key (תאריך תחילת היום הבא אחרי ה-12:00 הרלוונטי)
- התוויות "היום" / "אתמול" / "יום X • DD/MM" מחושבות ביחס לתחילת יום העבודה הנוכחי

### 4. השבוע (`draftsThisWeek`)
נשאר כפי שהוא — חלון 7 ימים אחורה הוא מטריקה גסה ולא מושפעת מהדיון.

## תוצאה צפויה לבוקר זה (יום שני 25/5)
- **Hands-Free Index — היום**: 12/12 (12 המוצרים מ-24/5 בערב נכללים)
- **כותרת קבוצה ראשונה**: "היום" — מציגה את 12 המוצרים שנסרקו אתמול בלילה
- בערב, אחרי 12:00, ה"יום" יתאפס ויחכה לסריקת הלילה הבאה

## הסבר טכני
```text
זמן נוכחי: יום ב', 25/5 09:00
workdayStart = 24/5 12:00  (כי 09:00 < 12:00, חוזרים יום אחורה)
מוצר שנוצר 24/5 18:42 → 18:42 > 12:00 → "היום"
מוצר שנוצר 23/5 16:37 → 16:37 ביום ראשון < workdayStart → "אתמול"

זמן נוכחי: יום ב', 25/5 14:00
workdayStart = 25/5 12:00  (כי 14:00 ≥ 12:00)
מוצר שנוצר 24/5 18:42 → < 25/5 12:00 → "אתמול"
```
